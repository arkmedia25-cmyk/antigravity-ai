1. Claude Code Skills Nedir?
Temel Kavramlar ve Mimari
Claude Code Skills (Agent Skills), Claude'un yeteneklerini modüler, tasinabilir paketlerle genisletir. Her
skill bir SKILL.md dosyasi etrafinda insa edilir: YAML frontmatter (ad + aciklama) ve Markdown
talimatlari. Claude, goreviyle en ilgili skill'i otomatik olarak secer ve devreye alir. Ayni skill Claude.ai,
Claude Code ve API uzerinde calisir.
Temel Ilke: Skills, Claude'u kendi basina yapamayacagi seyleri yapabilen veya sizin spesifik is akisinizi
kodlayan uzman asistanlara donusturur. Bir kez yaz, her yerde kullan.

2. Skill Creator — Mart 2026 Guncellemesi
Test, Olc, Iyilestir
Anthropic, Mart 2026'da Skill Creator'a yazilim gelistirme disiplinini getirdi. Artik kod yazmadan
skill'lerinizi test edebilir, kiyaslayabilir ve otomatik olarak optimize edebilirsiniz. Guncelleme Claude.ai,
Cowork ve Claude Code plugin olarak mevcuttur.
4 Calisma Modu
CREATE
Sifirdan skill insa et. Gereksinim toplama, SKILL.md taslagi, test case olusturma ve
iteratif iyilestirme dongusu.
EVAL
Skill'in gercekten calisip calismadigini olc. Her test case icin pass/fail orani, sure ve token
kullanimi raporlanir.
IMPROVE
Basarisiz test case'lerini analiz ederek skill govdesini veya description'i otomatik
gunceller.
BENCHMAR
K
Model guncellemelerinden once/sonra standart bir kiyaslama calistirir. CI pipeline'ina
entegre edilebilir.

Trigger Optimizasyon Dongusu (run_loop.py)
Description kalitesi, skill'in ne zaman tetiklenecegini belirleyen en kritik faktordu. Arastirmalar, optimize
edilmis description ile tetiklenme oraninin %20'den %90'a cikabildigini gosterdi. Otomasyon dongusu:
- Adim 1: Eval set'i %60 train / %40 held-out test olarak boler
 
- Adim 2: Mevcut description'i her query icin 3 kez test eder (guvenilirlik icin)
 
- Adim 3: Basarisiz olanlari analiz ederek yeni description onerir (max 200 kelime)
 
- Adim 4: Yeni description'i hem train hem test uzerinde degerlendirir
 
- Adim 5: 5 iterasyon boyunca tekrarlar; en iyi test skorunu secer (overfit onlenir)
 
- Adim 6: HTML raporu olusturur, best_description JSON olarak doner
 
Pratik Sonuc: Description bir 'metadata' degil, ogrenilir bir parametredir. Artik description'i tahmin
etmek yerine, test sonuclarina gore muhendislik yapabilirsiniz.
